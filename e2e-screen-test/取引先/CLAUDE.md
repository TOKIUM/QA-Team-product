# 取引先 E2Eテストプロジェクト

## 作業ルール（必ず守ること）

### タスク分割ルール
- **1回の応答では1タスクだけ実行する**。複数タスクの指示を受けた場合、最初のタスクのみ実行し、完了後に「次のタスクに進みますか？」とユーザーに確認する。
- タスクの区切りの目安:
  - ファイル1つの作成・修正 = 1タスク
  - テスト1スイートの実行・修正 = 1タスク
  - 画面1つの解析 = 1タスク
- 複数ファイルにまたがる変更でも、論理的に1つの変更であれば1タスクとして扱ってよい。

### コンテキスト節約ルール
- ANALYSIS_REPORT_*.md は**必要なセクションだけ**を読む。全体を一度に読まない。
- テストファイルの内容確認が必要な場合、対象ファイルだけを読む。関連ファイルを先回りで読まない。
- スクリーンショット（.png）は指示がない限り読み込まない。

## プロジェクト概要
TOKIUM 請求書発行（ステージング環境）の取引先E2Eテスト自動化。Playwright + Python (pytest) を使用。

- **対象URL**: `https://invoicing-staging.keihi.com/partners`
- **認証**: 環境変数 `TEST_EMAIL` / `TEST_PASSWORD`（.envは `../ログイン/.env` に配置）

## フォルダ構成

```
取引先/
├── CLAUDE.md                        # 本ファイル（セッション管理）
├── ANALYSIS_REPORT_PARTNERS.md      # 画面解析レポート
├── conftest.py                      # pytest共通fixture（logged_in_page）
├── pytest.ini                       # testpaths=.
└── test_partner_list.py             # 取引先一覧テスト（19テスト・全PASS）
```

## テスト実行方法

```bash
# 環境変数を設定してテスト実行
export TEST_EMAIL="ikeda_n+th1@tokium.jp"
export TEST_PASSWORD="Qa12345678"
cd 取引先/
python -m pytest test_partner_list.py -v --headed
```

## 画面構造（要約）

### 一覧画面 (/partners)
- ヘッダー: ロゴ、ユーザー名、ヘルプ、事業所切替（請求書画面と共通）
- サイドバー: 請求書、取引先、帳票レイアウト（請求書画面と共通）
- アクション: 「取引先を追加」ボタン、「その他の操作」（インポート/エクスポート）
- 検索フォーム: 送付方法、取引先コード/名、送付先担当者名、自社担当者名、メールアドレス、識別コード、住所、メモ、凍結CB、Web送付変更要求CB
- テーブル: 取引先コード/取引先名/送付先/送付先担当者名/自社担当者名/登録日/識別コード
- ページネーション: 表示件数（10/20/50/100）、前後ページ

### 取引先更新モーダル（行クリック時）
- タイトル: 「取引先更新」（article要素）
- 2タブ構成: 取引先項目 / Web送付設定
- 取引先項目タブ: 送付方法、住所、送付先担当者情報、自社担当者情報、基本情報（メモ・識別コード）
- Web送付設定タブ: 閲覧用メールアドレス管理（上限30件）
- ボタン: 保存 / キャンセル / 凍結する

### 取引先入力モーダル（新規追加時）
- タイトル: 「取引先入力」（dialog要素）
- タブなし（単一フォーム）
- 更新モーダルとの差分: 送付先メールアドレス（To/Cc）あり、識別コードなし、凍結ボタンなし

## 技術的制約・知見

### 検索フォームのポインタインターセプト
- 検索フォーム展開時にテーブル領域がインターセプトされる
- `click(force=True)` で検索ボタン・リセットボタンのクリックを回避
- モーダルを開く前に `close_search_form()` で検索フォームを閉じる

### ロケーター重複問題
- 「検索条件」ボタンと「検索条件を追加」が `name="検索条件"` でマッチ → `exact=True` で解決
- 「取引先コード」「取引先名」「メモ」「部署」「役職」がフォーム/モーダルで重複 → `form`/`article`でスコープ指定

### モーダルのDOM構造
- 更新モーダル: `article` 要素（`heading "取引先更新"` でフィルタ）
- 新規モーダル: `dialog` 要素（`heading "取引先入力"` でフィルタ）
- ×ボタン: `heading.locator(".. >> button").first` で取得

### ページネーションのDOM構造
- ボタンは `main > button` ではなく `div._pagingControl_ > button` にネスト
- 件数テキストの親要素 `.locator("..")` からbutton取得が確実
- 次ページ = `paging_container.locator("button").nth(1)`

### 送付方法の選択肢差分
- 検索フォーム: 全て/メール/Web送付/郵送代行/FAX送付/その他（6択）
- モーダル: メール/Web送付/その他（3択）
