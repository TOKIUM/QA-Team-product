## 7. 技術的制約・知見

### 7.1 ロケーター重複問題
以下の要素は検索フォームとモーダル内で同名のため、スコープ指定が必要：
- `textbox "取引先コード"` → 検索フォーム内 vs モーダル内
- `textbox "取引先名"` → 検索フォーム内 vs モーダル内
- `textbox "メモ"` → 検索フォーム内 vs モーダル内
- `textbox "部署"` → 送付先側 vs 自社側（モーダル内で2つ）
- `textbox "役職"` → 送付先側 vs 自社側（モーダル内で2つ）

**解決方法**: `form` や `dialog` でスコープを絞る、または `nth()` で位置指定

### 7.2 モーダルの種類判別
| 操作 | モーダルタイトル | DOM要素 |
|------|---------------|---------|
| 行クリック | 取引先更新 | `article` heading "取引先更新" |
| 「取引先を追加」ボタン | 取引先入力 | `dialog` heading "取引先入力" |

### 7.3 検索フォームの表示切替
- `button "検索条件"` クリックで検索フォームの展開/折りたたみ
- フォーム内の全フィールドはDOMに常に存在（visibility切替）

### 7.4 データ件数
- 現在1014件のテスト用取引先データが存在
- ページネーションは10/20/50/100件表示に対応

## 8. テスト設計方針

### 8.1 テストカテゴリ
| カテゴリ | テスト観点 | 優先度 |
|---------|-----------|--------|
| 一覧表示 | ページ読み込み、テーブル列確認、件数表示 | 高 |
| 検索 | 各フィールド検索、複合条件、リセット | 高 |
| ページネーション | ページ送り、表示件数切替 | 中 |
| 取引先追加 | 新規モーダル開閉、必須項目、保存 | 高 |
| 取引先更新 | 更新モーダル開閉、値変更、保存 | 高 |
| タブ切替 | 取引先項目 ↔ Web送付設定 | 中 |
| その他の操作 | メニュー表示確認 | 低 |
