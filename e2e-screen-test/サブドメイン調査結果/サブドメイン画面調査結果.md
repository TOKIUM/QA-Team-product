## 1. 画面遷移図

```
invoicing-staging.keihi.com/login             ← 起点
  │
  ├─[ログイン] → invoicing直接ログイン（サブドメイン不要）
  │
  ├─[TOKIUM IDでログイン] (href="/auth-redirect")
  │   │
  │   ▼ リダイレクト
  │
  dev.keihi.com/users/sign_in                 ← TOKIUM IDログイン画面
  │
  ├─[メール+PW入力 → ログイン]
  │   ├─ 成功 → invoicing SSOコールバック → /invoices
  │   └─ 失敗（サブドメイン必須アカウント）
  │       → ⚠「許可されていないログイン方法です。
  │           サブドメインを入力してからログインしてください。」
  │
  ├─[Googleでログイン] → POST /omniauth/google_oauth2
  ├─[Microsoft 365でログイン] → POST /omniauth/microsoft_office365
  ├─[パスワードを忘れた方はこちら] → /users/password/new
  │
  └─[サブドメインを入力]
      │
      ▼
  dev.keihi.com/subdomains/input              ← サブドメイン入力画面
      │
      ├─ 空欄 → 送信ボタン disabled（クリック不可）
      ├─ 無効値入力 → ⚠「無効なサブドメインです」
      ├─[ログイン画面に戻る] → dev.keihi.com/users/sign_in
      │
      └─ 有効値入力（例: th-01）→ [送信]
          │
          ▼
  th-01.dev.keihi.com/subdomains/sign_in      ← サブドメインログイン画面
      │
      ├─[メール+PW入力 → ログイン]
      │   └─ 成功 → th-01.dev.keihi.com/transactions
      │              フラッシュ:「ログインしました。」
      │
      ├─[パスワードを忘れた方はこちら]
      │
      └─[サブドメイン入力に戻る] → dev.keihi.com/subdomains/input
```
