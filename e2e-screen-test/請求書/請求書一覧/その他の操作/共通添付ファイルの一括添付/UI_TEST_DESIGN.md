# 共通添付ファイルの一括添付 UIテスト設計書

**対象機能**: 「その他の操作」→「共通添付ファイルの一括添付」モーダル
**対象URL**: https://invoicing-staging.keihi.com/invoices
**作成日**: 2026-02-17
**テスト種別**: 手動テスト（目視確認）
**前提**: 自動テスト50件（カテゴリA〜K）は全PASSの状態

---

## 1. テスト設計方針

### UIテストの目的
自動テストではカバーできない **視覚的品質** と **ユーザビリティ** を人の目で検証する。

### カバー範囲
自動テストとの役割分担:

| 観点 | 自動テスト（実装済み） | UIテスト（本設計） |
|------|---------------------|------------------|
| DOM構造・属性 | ✅ カテゴリJ | - |
| 画面遷移・状態 | ✅ カテゴリH,I | - |
| バリデーション動作 | ✅ カテゴリB,C,D,K | - |
| 視覚的デザイン | - | ✅ レイアウト・色・フォント |
| アニメーション | - | ✅ トランジション・ローディング |
| レスポンシブ | - | ✅ ウィンドウリサイズ対応 |
| アクセシビリティ | 一部（role属性） | ✅ フォーカス・キーボード操作 |
| テキスト品質 | 一部（内容一致） | ✅ 誤字脱字・翻訳・表現の統一 |
| エラーの視認性 | 一部（存在確認） | ✅ 色のコントラスト・位置 |
| ドラッグ&ドロップ | - | ✅ 操作の自然さ・フィードバック |

## 5. 判定基準

### 重要度レベル

| レベル | 定義 | 対応 |
|--------|------|------|
| Critical | 機能が使用不能になるUI不具合 | 即修正必須 |
| Major | 操作に支障があるが回避策がある | 次回リリースまでに修正 |
| Minor | 見た目の軽微な不具合 | バックログに追加 |
| Cosmetic | デザインの好みレベル | 改善提案として記録 |

### 各カテゴリの期待重要度

| カテゴリ | 想定される不具合の重要度 |
|---------|---------------------|
| UI-1 ステップウィザード | Minor〜Major（進行状況がわからないとUX低下） |
| UI-2 Step1 | Minor〜Major（レイアウト崩れは操作に影響） |
| UI-3 Step2 | Minor〜Major（テーブル崩れは誤操作の原因） |
| UI-4 Step3 | Minor〜Cosmetic（完了は機能的には問題なし） |
| UI-5 エラー | Major〜Critical（エラーが見えないと誤操作の原因） |
| UI-6 レスポンシブ | Minor〜Major（低解像度ユーザーに影響） |
| UI-7 アクセシビリティ | Minor〜Major（キーボード操作不可はアクセシビリティ違反） |
| UI-8 アニメーション | Cosmetic〜Minor（UXの滑らかさ） |
| UI-9 テキスト | Minor（誤字脱字は信頼性に影響） |
| UI-10 D&D | Minor〜Major（D&Dが動作しないと代替手段が必要） |
