# 共通添付ファイルの一括添付 テスト設計書

**対象機能**: 「その他の操作」→「共通添付ファイルの一括添付」モーダル
**対象URL**: https://invoicing-staging.keihi.com/invoices
**作成日**: 2026-02-17
**前提**: 請求書一覧画面でチェックボックス1件以上選択済み

---

## 1. 機能仕様サマリー

### 1-1. 画面フロー

```
一覧画面
  └─ チェックボックス選択（1件以上）
      └─ 「その他の操作」→「共通添付ファイルの一括添付」
          └─ モーダル（3ステップウィザード）
              ├─ Step 1: ファイル選択
              │   ├─ 「新規アップロード」タブ（デフォルト）
              │   │   ├─ ドロップゾーン / ファイル選択ボタン
              │   │   ├─ input[type="file"] (multiple=true, accept制限なし)
              │   │   └─ 右ペイン: 選択済みファイル一覧
              │   └─ 「既存から選択」タブ
              ├─ Step 2: 確認画面
              │   ├─ 添付するファイル(N件) 表示
              │   ├─ 添付先の帳票一覧テーブル（取引先/送付方法/請求書番号/金額/判定）
              │   ├─ 非同期判定: 「添付可否を判定中...」→ 判定完了
              │   └─ フッター: 「添付を実行可能です」or エラーメッセージ
              └─ Step 3: 完了画面
                  ├─ 「添付が完了しました」
                  ├─ 「共通添付ファイルが N 件の請求書に添付されました」
                  └─ 「閉じる」ボタン
```

### 1-2. バリデーション仕様（2段階）

| 段階 | タイミング | 判定元 | 条件 | UI挙動 |
|------|-----------|--------|------|--------|
| Step 1 バリデーション | ファイルセット直後 | フロントエンド | 1ファイル10MB超過 | ファイル横「エラー」、確認へ進むdisabled |
| Step 1 バリデーション | ファイルセット直後 | フロントエンド | 拡張子なし | ファイル横「エラー」、確認へ進むdisabled |
| Step 2 バリデーション | 確認へ進むクリック後 | サーバー側非同期 | ファイル数11件以上 | 判定列「添付数上限を超過しています」 |
| Step 2 バリデーション | 確認へ進むクリック後 | サーバー側非同期 | 合計サイズ上限超過 | 判定列「ファイルサイズ上限を超過しています」 |
| Step 2 バリデーション | 確認へ進むクリック後 | サーバー側非同期 | 既添付と合計で上限超過 | 判定列「添付数上限を超過しています」 |

### 1-3. 制限値

| 項目 | 上限 | 備考 |
|------|------|------|
| 1ファイルサイズ（フロント） | 10MB | 10.1MB(10,590,617B)でStep1ブロック |
| 合計サイズ（サーバー） | ~9.5MB実効 | 9.9MB(10,380,902B)でStep2NG, 5.1MB(5,347,737B)はOK |
| ファイル数 | 10件 | 11件指定→10件のみ受付、Step2で「添付数上限超過」 |
| 既存添付との合計件数 | 10件 | 既に添付がある請求書は累積カウント |
