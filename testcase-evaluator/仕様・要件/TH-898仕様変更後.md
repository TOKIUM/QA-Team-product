#　ログイン方法でID・PWとTOKIUM IDが存在し、TOKIUM IDだけ利用したい場合にフラグでログイン制限できるように

#　主要な詳細
##　完了の定義
+ 事業所単位のフィーチャーフラグとして「TOKIUM IDでログインのみ使用する」を裏側で設定できるようにする。
    * 単純な真偽値のフラグではなく、有効化開始日みたいなカラムとする。終了日はなし
+ 「TOKIUM IDでログインのみ使用する」の場合、ログイン時にID/PWでログインしようとするとエラーにする。
    * すでにID/PWでログインしている人は特に強制ログアウトなどはさせない。
+ ただし、TOKIUM管理ユーザー（th+〜@tokium.jpで終わるもの）は制限の対象外とする。

## 説明
元チケットからのまとめ

+ フラグをONにした際、既にID/PWでログイン済みのユーザーのセッションの扱い
    * 次回ログイン時から適用 ✅️
+ ID/PWログイン中に「TOKIUM IDのみ ON」の事業所へ切り替える場合の挙動
    * 切り替えできる ✅️
+ TOKIUM IDログインのみ許可するかどうかはTHの環境作成（ユーザーログイン前）の段階で決まるのか、後から随時ON/OFFの変更をしたいのか
    * どちらもありうる ✅️

# TOKIUM IDでのログインのみ許可する設定 - 仕様説明
## この機能でできること
事業所ごとに「TOKIUM IDでのログインのみ許可する」設定を有効にすることで、メールアドレス＋パスワードによるログインを制限できます。セキュリティ強化のため、TOKIUM IDでのログインに一本化したい事業所向けの機能です。

# 基本的な動作
## 設定が無効の場合（従来通り）
+ メールアドレス＋パスワードでログインできる
+ TOKIUM IDでログインできる
+ どちらの方法でもログイン可能（変更なし）

# 設定が有効の場合
+ メールアドレス＋パスワードでのログインが拒否される
+ TOKIUM IDでのログインは引き続き可能
+ 拒否時のエラーメッセージ：「この事業所ではTOKIUM IDでのログインのみ許可されています。TOKIUM IDでログインしてください。」

# 設定方法
+ 事業所ごとに「有効化開始日」を設定する
+ 開始日以降、設定が自動的に有効になる
+ 開始日が未来の日付であれば、その日までは従来通りメールアドレス＋パスワードでもログイン可能
+ 開始日を未設定にすれば、制限は無効（従来通り）

## 制限されるもの
+ 操作：メールアドレス＋パスワードでのログイン 
+ 制限あり？：制限される
+ 説明：エラーメッセージが表示されログイン不可 

## 制限されないもの
+ 操作：TOKIUM IDでのログイン 
+ 制限あり？：制限なし 
+ 説明：常にログイン可能 

+ 操作：ログイン済みユーザーの操作  
+ 制限あり？：制限なし
+ 説明：設定有効化前にログインしていた場合、セッションはそのまま維持される

+ 操作：テナント（事業所）切替 
+ 制限あり？：制限なし 
+ 説明：ログイン済み状態での事業所切替は制限を受けない

# TOKIUM管理ユーザーについて
TOKIUM管理ユーザー（th+〜@tokium.jp）は、この設定の制限対象外です。設定が有効な事業所であっても、メールアドレス＋パスワードでログインできます。これは運用・サポート対応のためです。

# 複数事業所に所属するユーザーの場合
同じメールアドレスで複数の事業所に所属しているユーザーの場合、最初に登録された事業所の設定が適用されます。

例1
事業所：事業所 A 
登録順：1番目（最古）
設定：有効 
ログイン結果：ログイン拒否（事業所 A の設定が適用される） 

事業所：事業所 B 
登録順：2番目 
設定：無効 
ログイン結果：-

例2
事業所：事業所 A  
登録順：1番目（最古）
設定：無効 
ログイン結果：ログイン成功（事業所 A の設定が適用される） 

事業所：事業所 B 
登録順：2番目
設定：有効 
ログイン結果：-

補足: ログイン時は最初に登録された事業所のユーザーとして認証されるため、その事業所の設定のみが判定に使われます。

# 設定有効化のタイミングについて
+ 設定の有効化は新規ログイン時のみチェックされます
+ 既にログイン中のユーザーには即座に影響しません
+ 次回ログイン時から制限が適用されます

# タイムライン例
2/1: 事業所の管理者が「有効化開始日 = 2/10」で設定  ↓
2/1〜2/9: 従来通り、メールアドレス＋パスワードでもログイン可能  ↓
2/10〜: メールアドレス＋パスワードでのログインが拒否される
        （TOKIUM IDでのログインのみ可能）

# 簡易FAQ
Q: 設定を有効にした瞬間、ログイン中のユーザーは強制ログアウトされますか？
A: いいえ。既存のセッションは維持されます。次回ログイン時から制限が適用されます。

Q: TOKIUM IDを持っていないユーザーはどうなりますか？
A: メールアドレス＋パスワードでのログインが拒否されるため、ログインできなくなります。設定を有効にする前に、対象事業所の全ユーザーがTOKIUM IDでログインできる状態になっていることを確認してください。

Q: 設定を無効に戻すことはできますか？
A: はい。有効化開始日を未設定にすれば、従来通りメールアドレス＋パスワードでのログインが可能に戻ります。

Q: テナント切替で制限のある事業所に移動できますか？
A: はい。テナント切替はこの制限の対象外です。ログイン済みの状態であれば、制限のある事業所にも切り替え可能です。
