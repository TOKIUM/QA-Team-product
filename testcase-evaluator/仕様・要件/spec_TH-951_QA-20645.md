## 対応概要

セキュリティ検証で、パスワードリセットや新規登録の確認メールに記載されるURLを細工することで、ユーザーを外部の悪意あるサイトに誘導できる問題（オープンリダイレクト）が見つかりました。

本修正では、確認メールからの遷移先をTH自身のサイトのみに限定し、外部サイトへの遷移を禁止する対策を行いました。通常の操作フローには影響ありません。

また、この修正は **TH / WDL** 両方とも適用されます。

## 影響範囲

以下の認証フローが影響範囲です。

| フロー | 影響内容 |
| --- | --- |
| パスワードリセット（TH / WDL） | 「パスワードを忘れた場合」からメール送信 → メール記載URLからパスワード再設定ページへの遷移 |
| 新規登録（TH / WDL） | 「新規登録はこちら」からユーザー情報入力・登録 → 確認メールの送信 |
| メール確認（TH / WDL） | 新規登録後の確認メール記載URL押下 → 登録完了ページへの遷移 |
| アカウントロック解除（TH / WDL） | ログイン試行を5回以上失敗 → ロック解除メール記載URL押下 → ロック解除完了ページへの遷移 |
| TOKIUM IDでログイン（THのみ） | TOKIUM IDで認証後、THに戻ってログインが完了するまでの処理 |

※ アカウントロック解除とTOKIUM IDでログインはオープンリダイレクト脆弱性の対象外ですが、防御的に同じ制限を適用しています。

## テスト観点（リグレッション確認）

**TH / WDL両方**で、各フローが修正前と同様に正常動作することを確認してください。

- **パスワードリセット**: 「パスワードを忘れた場合」→ メールアドレス入力 → メール送信 → メール記載URL押下 → パスワードリセットページに遷移し、パスワードを再設定できること
- **新規登録**: 「新規登録はこちら」→ ユーザー情報入力 → 登録 → 確認メール記載URL押下 → 登録完了ページに遷移すること
- **アカウントロック解除**: ログイン試行を5回以上失敗 → ロック解除メール記載URL押下 → ロック解除完了ページに遷移すること
- **（THのみ）TOKIUM IDでログイン**: TOKIUM IDでの認証後、正常にTHにログインできること

※ 脆弱性修正の検証（URL改ざんによる外部リダイレクトの拒否）は自動テストでカバー済みです。
