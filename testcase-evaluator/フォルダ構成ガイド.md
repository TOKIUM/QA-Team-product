# テストケース評価ツール フォルダ構成ガイド

テスト項目書の自動評価・生成ツール一式の構成ガイドです。

---

## フォルダ全体像

```
testcase-evaluator/
│
├── フォルダ構成ガイド.md        <- この文書
│
├── 仕様・要件/                  インプット（仕様書・チケット情報）
├── 改善指示/                    インプット（改善方針・運用ルール）
├── 参考例/                      インプット（レビュー済みの良い例 62件）
├── テスト項目書テンプレート/    インプット（Excel出力テンプレート）
│
├── 評価ツール/                  汎用ツール（評価・生成・学習）
├── 観点評価ツール/              汎用ツール（観点シート評価）
├── 品質基準/                    テスト品質の統一基準（Single Source of Truth）
│
└── 修正スクリプト/              TH-898専用（修正パターンの参考）
```

---

## 各フォルダの詳細

### 仕様・要件/
テスト対象機能の仕様書・チケット情報を格納。
- `TH-898.md`, `TH-898_ログイン機能仕様書.docx` — TH-898関連
- `spec_TH-951_QA-20645.md` — TH-951+QA-20645統合仕様
- `ガードレール/` — QAリーダー制約資料

### 改善指示/
テスト項目書の改善方針・運用ルールを格納。
- `運用ルール.png` — 評価の最上位基準
- `改善要望.md`, `改善方針.md`

### 参考例/
レビュー済みの良いテスト項目書（62件）。学習・粒度比較に使用。
- `参考_品質混在/` — 品質がばらつく例（学習の参考）
- 追加するほど `benchmark.json` の精度が向上

### 評価ツール/（コア）
プロジェクト横断で再利用可能な汎用ツール。
| ファイル | 役割 |
|---------|------|
| `evaluate_testcases.py` | 100点満点の自動評価（6チェック項目） |
| `generate_testcases.py` | 仕様書からテストケース自動生成 |
| `learn_benchmark.py` | 参考例から基準値を学習 |
| `common.py` | 共通ユーティリティ |
| `benchmark.json` | 学習済み基準値 |
| `ng_words_sample.txt` | NGワードサンプル |
| `テストケース評価ガイド.md` | 使い方+アーキテクチャ |
| `AI向け評価ツール利用ガイド.md` | Claude向け実行指示 |

### 観点評価ツール/
観点シートの自動評価（5カテゴリ×20点、計100点）。テストケース評価と併用。

### 品質基準/（Single Source of Truth）
テスト品質に関する基準を一元管理。評価ツール・生成ツール・Skillはすべてここを参照。
| ファイル | 内容 |
|---------|------|
| `README.md` | 全体マップ・更新ルール |
| `記述ルール.md` | 書式・表現・フォーマットの統一基準 |
| `観点チェックリスト.md` | 16機能+7非機能テスト観点の判定基準 |
| `テストパターン.md` | 6つの具体的テスト設計パターン |
| `評価基準.md` | 自動評価の6チェック項目+採点ロジック |
| `改善ログ.md` | 評価・レビューで得た知見の蓄積 |

### 修正スクリプト/
TH-898専用の修正スクリプト。他プロジェクトでは直接再利用しないが、修正パターンの参考として保管。
- `fix_v7*.py` — テストケース・観点シートの一括修正
- `compare_excel.py` — Excel比較分析

---

## 作業の流れ

```
仕様書/チケット ──→ generate_testcases.py ──→ テスト項目書.xlsx
                                                    │
参考例/ ──→ learn_benchmark.py ──→ benchmark.json    │
                                        │           ▼
                              evaluate_testcases.py ──→ 評価レポート（100点）
                              + 観点評価ツール ──→ 観点評価レポート（100点）
                                        │
                                  修正 → 再評価のサイクル
```
