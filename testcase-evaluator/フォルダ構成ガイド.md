# ログイン情報 フォルダ構成ガイド

TH-898（ログイン制限機能）のテスト項目書を改善するプロジェクトで使用した全資料の整理・管理ガイドです。

---

## フォルダ全体像

```
ログイン情報/
│
├── フォルダ構成ガイド.md        ← この文書
│
├── 仕様・要件/                  ① インプット（何を作るか）
├── 画面キャプチャ/              ② インプット（どんな画面か）
├── 改善指示/                    ③ インプット（どう直すか）
├── 参考例/                      ④ インプット（お手本）
│
├── 成果物/                      ⑤ アウトプット（最終版）
├── 修正スクリプト/              ⑥ アウトプット（修正に使ったツール）
│
├── 評価ツール/                  ⑦ 汎用ツール（他プロジェクトでも再利用可能）
│
└── archive/                     ⑧ 過去の作業履歴
```

---

## 各フォルダの詳細

### ① 仕様・要件/

| 項目 | 内容 |
|---|---|
| **目的** | テスト対象機能の仕様・要件を格納する |
| **いつ使うか** | テストケース作成・レビュー時に仕様と突き合わせる |
| **誰が更新するか** | PM・開発者がチケット起票時に配置 |

| ファイル | 説明 |
|---|---|
| `TH-898.md` | チケットの対応情報メモ（背景・方針） |
| `TH-898_ログイン機能仕様書.docx` | ログイン機能の正式な仕様書 |
| `ログイン情報.md` | ログイン関連の基本情報メモ |

**ポイント**: 評価ツールにNGワードを指定する際、ここの仕様書に記載された正式名称を基準にする。

---

### ② 画面キャプチャ/

| 項目 | 内容 |
|---|---|
| **目的** | テスト対象画面のスクリーンショットを格納する |
| **いつ使うか** | テストケースの手順・期待値を書くときに画面の正確な文言を確認する |
| **誰が更新するか** | QA担当者がテスト設計前にキャプチャを取得 |

```
画面キャプチャ/
└── ログイン/
    ├── ログイン.png                          ← メインのログイン画面
    ├── TOKIUM IDでログイン/
    │   └── ログインはこちら.png              ← TOKIUM IDログイン画面
    ├── パスワードを忘れた場合/
    │   └── パスワード再設定.png              ← パスワード再設定画面
    └── 新規登録はこちら/
        └── ユーザー登録.png                  ← ユーザー登録画面
```

**ポイント**: テストケースの期待値に書く画面文言（ボタン名、エラーメッセージ等）は、ここのキャプチャと仕様書から正確に転記する。

---

### ③ 改善指示/

| 項目 | 内容 |
|---|---|
| **目的** | テスト項目書の改善方針・チェック基準を格納する |
| **いつ使うか** | テスト項目書をレビュー・修正するときの指針として参照 |
| **誰が更新するか** | QAリーダー・マネージャーが方針を策定時に配置 |

| ファイル | 説明 |
|---|---|
| `運用ルール.png` | テスト項目書の運用ルール（公式基準）。評価ツールのチェック項目はこれが根拠 |
| `改善要望.md` | 具体的な改善要望の一覧 |
| `改善方針.md` | 改善の進め方・優先順位 |

**ポイント**: `運用ルール.png` が評価の最上位基準。評価ツールの減点ロジックはすべてここに基づいている。

---

### ④ 参考例/

| 項目 | 内容 |
|---|---|
| **目的** | レビュー済みの良いテスト項目書を格納する（学習・比較用） |
| **いつ使うか** | 評価ツールの学習（ベンチマーク生成）、新しいテストケースの粒度比較 |
| **誰が更新するか** | レビュー通過したテスト項目書を随時追加 |

| ファイル | 説明 |
|---|---|
| `テスト項目書_取引先マスタのテーブル構造変更（企業マスタ追加）.xlsx` | 大規模機能追加の参考例（6,077件） |
| `テスト項目書_事前申請の更新で...xlsx` | バグ修正の参考例（38件） |

**使い方**:
```bash
# このフォルダの参考例から基準値を学習
python 評価ツール/learn_benchmark.py 参考例/ --output 評価ツール/benchmark.json
```

**ポイント**: 参考例を追加するほど評価の精度が上がる。新しい案件でレビュー通過したテスト項目書があれば、ここにコピーして `learn_benchmark.py` を再実行する。

---

### ⑤ 成果物/

| 項目 | 内容 |
|---|---|
| **目的** | 最終的に提出・使用するテスト項目書を格納する |
| **いつ使うか** | テスト実行時、レビュー提出時 |
| **誰が更新するか** | QA担当者が修正完了後に配置 |

| ファイル | 説明 |
|---|---|
| `テスト項目書_改善後_v7.xlsx` | 最終版（214件、評価100/100点） |
| `テスト項目書_原本.xlsx` | 改善前の原本（比較用に保存） |

**ポイント**: 修正前後の差分を確認したい場合は、原本とv7を並べて比較する。

---

### ⑥ 修正スクリプト/

| 項目 | 内容 |
|---|---|
| **目的** | テスト項目書をプログラムで一括修正するために使ったスクリプトを格納する |
| **いつ使うか** | 同じパターンの修正を大量に適用する必要があるとき |
| **誰が更新するか** | AI（Claude Code）が修正依頼時に自動生成 |

| ファイル | 説明 |
|---|---|
| `fix_v7.py` | v6→v7 の主要修正（我流単語置換、複数こと分離等） |
| `fix_v7_add_testcases.py` | 不足テストケースの追加 |
| `fix_v7_final.py` | 最終仕上げ（残存NGワード修正） |
| `fix_v7_kantensheet.py` | 観点シートの修正 |
| `fix_v7_kanten_add2.py` | 観点シートへの観点追加 |

**ポイント**: これらは TH-898 専用のスクリプト。他プロジェクトではAIが新たに生成するため、直接再利用はしない。修正の進め方の参考として残している。

---

### ⑦ 評価ツール/

| 項目 | 内容 |
|---|---|
| **目的** | テスト項目書を自動評価・学習するための汎用ツール一式 |
| **いつ使うか** | どのプロジェクトのテスト項目書でも評価したいとき |
| **誰が更新するか** | チェック項目を追加・変更したいときに編集 |

| ファイル | 種別 | 説明 |
|---|---|---|
| `evaluate_testcases.py` | スクリプト | 100点満点の自動評価（5チェック項目） |
| `learn_benchmark.py` | スクリプト | 参考例から基準値を学習して `benchmark.json` を生成 |
| `benchmark.json` | データ | 学習済みの基準値（参考例2件から算出） |
| `ng_words_sample.txt` | データ | NGワードのサンプル（TSV形式） |
| `テストケース評価ガイド.md` | ドキュメント | 人間向けの評価手順・使い方ガイド |
| `AI向け評価ツール利用ガイド.md` | ドキュメント | Claude Codeに評価させるための設定・使い方 |
| `学習と評価の仕組み.md` | ドキュメント | 学習→評価パイプラインのアーキテクチャ |

**基本的な使い方**:
```bash
# 評価のみ
python 評価ツール/evaluate_testcases.py テスト項目書.xlsx

# NGワード付き評価
python 評価ツール/evaluate_testcases.py テスト項目書.xlsx --keywords ngwords.txt

# 参考例を学習してから評価
python 評価ツール/learn_benchmark.py 参考例/ --output 評価ツール/benchmark.json
python 評価ツール/evaluate_testcases.py テスト項目書.xlsx --keywords ngwords.txt
```

**ポイント**: このフォルダのツールはプロジェクト横断で再利用できる。NGワードだけプロジェクトごとに変える。

## フォルダ間の関係図

```
【インプット】                    【処理】                    【アウトプット】

 仕様・要件/ ──────┐
                    │
 画面キャプチャ/ ──┤
                    ├──→ 評価ツール/ ──→ 評価レポート（点数＋改善点）
 改善指示/ ────────┤         │
                    │         │
 参考例/ ──────────┘         ↓
    │                    修正スクリプト/ ──→ 成果物/
    │                                        （改善後のテスト項目書）
    ↓
 learn_benchmark.py
    ↓
 benchmark.json（学習済み基準値）
```

### 作業の流れ

1. **準備**: 仕様書・画面キャプチャ・改善指示を各フォルダに配置
2. **学習**（初回 or 参考例追加時）: `learn_benchmark.py` で参考例から基準値を生成
3. **評価**: `evaluate_testcases.py` でテスト項目書を100点満点で自動チェック
4. **修正**: 評価結果に基づき修正スクリプトを作成・実行
5. **完了**: 修正後のテスト項目書を成果物/に配置、旧版はarchive/に移動

## 別プロジェクトで同じ構成を使う場合

新しいプロジェクト（例: 請求書機能）でも同じフォルダ構成を作れます。

```
請求書機能/
├── 仕様・要件/          ← 請求書機能の仕様書を配置
├── 画面キャプチャ/      ← 請求書画面のキャプチャを配置
├── 改善指示/            ← 改善方針があれば配置
├── 参考例/              ← ログイン情報/参考例/ と共有可能
├── 成果物/              ← 完成したテスト項目書を配置
├── 修正スクリプト/      ← AIが自動生成
├── 評価ツール/          ← ログイン情報/評価ツール/ からコピー or シンボリックリンク
└── archive/             ← 過去バージョンを退避
```

評価ツールは共通で使えるため、コピーするか、パスを直接指定して実行すればOKです。
