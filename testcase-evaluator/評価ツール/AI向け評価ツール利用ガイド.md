## 1. 仕組みの概要

```
┌──────────────────────────────────────────────────┐
│  あなた（ユーザー）                                │
│  「このテスト項目書を評価して」                      │
└────────────────────┬─────────────────────────────┘
                     ↓
┌──────────────────────────────────────────────────┐
│  Claude Code                                      │
│                                                    │
│  ① CLAUDE.md を自動読み込み                        │
│     → 評価スクリプトの場所・チェック項目を把握       │
│                                                    │
│  ② evaluate_testcases.py を実行                    │
│     → 100点満点のスコア＋違反箇所を検出             │
│                                                    │
│  ③ 結果を報告 + 修正推奨を提示                      │
└──────────────────────────────────────────────────┘
```

### Claude Codeが自動で読むファイル

| ファイル | いつ読まれるか | 内容 |
|---|---|---|
| `~/.claude/CLAUDE.md` | **全チャット**の開始時 | 評価スクリプトのパス・チェック項目・評価手順 |
| `memory/MEMORY.md` | **同じフォルダ**からのチャット開始時 | テスト項目書の書き方・評価ロジックへのリンク |

つまり、**どのフォルダから新しいチャットを開いても**、Claudeは評価ツールの存在と使い方を知っています。

## 3. 基本の使い方

### 最もシンプルな依頼（1行だけ）

新しいチャットを開いて、以下を送信するだけです。

```
このテスト項目書を評価して
C:\Users\池田尚人\ClaudeCode用\テストケース評価自動化\成果物\テスト項目書_改善後_v7.xlsx
```

Claudeは自動で `evaluate_testcases.py` を実行し、100点満点のスコアと改善点を報告します。

> **注意**: NGワードを指定しない場合、「我流の単語」チェックはスキップされます。

### NGワード付きの依頼（推奨）

```
このテスト項目書を評価して

ファイル: C:\Users\池田尚人\ClaudeCode用\テストケース評価自動化\成果物\テスト項目書_改善後_v7.xlsx

NGワード:
- ID/PW → メールアドレス・パスワード
- PW → パスワード
- リカバリー → パスワード再設定
```

Claudeがこの情報からNGワードファイルを一時作成し、4項目すべてをチェックします。

### 評価後に修正もさせたい場合

```
修正してください
```

と追加で送信すれば、Claudeが修正スクリプトを作成・実行します。

## 5. プロンプト例集（学習）

学習とは、レビュー済みの良いテスト項目書（参考例）をAIに分析させ、「良いテストケースの基準値（ベンチマーク）」を自動生成することです。
学習によって生成された `benchmark.json` は、評価時のチェック5（粒度・品質）の判定基準として自動的に使われます。

### 学習の仕組み（概要）

```
参考例/ フォルダに良いテスト項目書.xlsx を配置
        ↓
learn_benchmark.py が全ファイルを分析
  - 期待値の文字数分布（平均・5%tile・95%tile）
  - 手順記載率・番号付きステップ率
  - 「こと」1回率・曖昧表現率
  - 良好な末尾表現パターンTOP15
        ↓
benchmark.json に基準値を保存
        ↓
evaluate_testcases.py が自動読み込みして評価に使用
```

> **参考例を追加して再学習するだけで、評価の精度が上がります。**

---

### 例L1: 最もシンプルな学習（参考例フォルダを指定するだけ）

```
参考例フォルダからベンチマークを学習してください。

参考例フォルダ: C:\Users\池田尚人\ClaudeCode用\テストケース評価自動化\参考例\
```

→ `learn_benchmark.py` を実行し、`benchmark.json` を生成・更新します。
→ 参考例フォルダ内のすべての `.xlsx` ファイルが分析対象になります。

### 例L2: 学習結果の確認

```
参考例フォルダからベンチマークを学習して、学習結果の内容を説明してください。

参考例フォルダ: C:\Users\池田尚人\ClaudeCode用\テストケース評価自動化\参考例\
```

→ 学習実行後、`benchmark.json` の中身をわかりやすく説明します。
→ 「参考例は何件か」「期待値の平均は何文字か」「良好な末尾表現は何か」等を報告。

### 例L3: 新しい参考例を追加して再学習

```
参考例フォルダに新しいファイルを追加したので、ベンチマークを再学習してください。

参考例フォルダ: C:\Users\池田尚人\ClaudeCode用\テストケース評価自動化\参考例\
```

→ 参考例フォルダに新しい `.xlsx` を手動で追加した後に、この命令を送ります。
→ 前回の学習結果は上書きされ、新しいファイルも含めた基準値に更新されます。

### 例L4: 特定のファイルを参考例に追加して再学習


## 6. 別プロジェクトで使う場合

1. **新しいNGワードリストを用意する**
   - 新しいプロジェクトの仕様書・PRDを確認
   - テストケースで使うべきでない略語・造語をリストアップ
   - チャット内でClaudeに伝えるか、TSVファイルとして保存

2. **チャットで依頼するだけ**
   - 評価スクリプトは共通で使える（シート名・カラムは自動検出）
   - NGワードだけプロジェクトごとに変える

## 7. 設定ファイルの役割と編集方法

### CLAUDE.md（全チャット共通の指示）

**場所**: `C:\Users\池田尚人\.claude\CLAUDE.md`

**役割**: Claudeが全てのチャットの冒頭で自動的に読み込む指示書。
テストケース評価の依頼を受けた時に何をすべきかが書かれている。

**編集が必要になるケース**:
- 評価スクリプトの保存場所を変えた時
- チェック項目を追加・変更したい時
- 評価フローの手順を変えたい時

**編集方法**: テキストエディタ（メモ帳、VS Code等）で直接開いて編集

### evaluate_testcases.py（評価スクリプト本体）

**場所**: `C:\Users\池田尚人\ClaudeCode用\テストケース評価自動化\評価ツール\evaluate_testcases.py`

**編集が必要になるケース**:
- 曖昧表現のNGパターンを追加したい時（`VAGUE_PATTERNS` リスト）
- 減点の配分を変えたい時（冒頭の定数）
- カラム自動検出のキーワードを追加したい時（`detect_columns` 関数）

### NGワードファイル（プロジェクト個別）

**場所**: プロジェクトごとに任意の場所に作成

**形式**: TSV（タブ区切り）
```
# コメント行

```

## 8. 評価の精度を上げるコツ

### NGワードは仕様書から拾う

仕様書を開き、以下を確認:
- 機能の正式名称（略語がテストケースで使われていないか）
- 画面名・ボタン名の正式名称
- エラーメッセージの正式文言

### 仕様書も一緒に渡す

仕様書のパスを伝えると、Claudeが:
- テストケースの網羅性を仕様書と突き合わせて確認
- 仕様書にある用語でNGワードを自動で提案
- 仕様にあるがテストケースにない項目を指摘

### 参考例も渡す

過去の良いテスト項目書のパスを伝えると、Claudeが:
- 粒度の比較を行い、粗すぎ・細かすぎを指摘
- 手順の書き方のスタイルを合わせるよう提案

## 9. 制限事項と注意点

### 自動チェックの限界

| 検出できる | 検出できない |
|---|---|
| 「正常であること」等の定型NGワード | 文脈に依存する曖昧さ（「正しいメールアドレスを入力→ログインできること」は適切） |
| 「こと」の出現回数 | 「こと」が2回でも正当なケース（例: 「〜すること：\n「〜〜〜〜」」は引用内の「こと」） |
| NGワード辞書に登録された単語 | 辞書に登録していない我流単語 |
| グループ全体が手順空欄 | 手順の詳しさが十分かの判断 |

→ **自動チェック（60-70%の品質保証）+ 手動チェック（残り30-40%）の組み合わせが最適**

### Excelファイルを開いたまま実行しない

Excelでファイルを開いていると `PermissionError` になります。
評価前にExcelを閉じてください。

### NGワードの部分一致に注意

`PW` をNGワードにすると「パスワード」内の「パスPW」にも一致する可能性があります。
誤検出が多い場合はClaudeに「PW は パス の直後を除外して」と伝えてください。
