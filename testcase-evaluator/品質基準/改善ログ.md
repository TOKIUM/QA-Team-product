# 改善ログ

テストケースの評価・レビュー・生成を通じて得た知見を蓄積します。
定期的にルール化できるものは `記述ルール.md` や `評価基準.md` に昇格させます。

---

## 記録フォーマット

```
### YYYY-MM-DD: [タイトル]
- **発見元**: 評価結果 / レビュー指摘 / 生成時の気付き
- **内容**: 具体的な問題と対応
- **ルール化**: 済 / 未（済の場合は昇格先を記載）
```

---

### 2026-03-12: 参考例3件から6つの弱点を特定
- **発見元**: DEVSEC-133(オープンリダイレクタ), DEVSEC-141(システム情報の表示), ログイン制限フラグ の修正済み参考例との比較
- **内容**:
  1. **観点シートの階層構造不足**: 確認すること①→②→画面→確認対象→詳細①→詳細②の5段階が必要。フラットなリストはNG
  2. **デシジョンテーブル未使用**: 条件の掛け合わせ（ログイン制限: 6条件×24パターン）はDT形式で網羅性を可視化すべき
  3. **テストシナリオ(開発側連携)シート欠落**: セットアップ用Railsコマンド+使用ユーザー区分+TC紐づけを含むシナリオシートが必要
  4. **期待値の分割不足**: 1操作に対し「成功すること」「画面遷移すること」「エラーメッセージが表示されないこと」を別TCに分割すべき
  5. **テスト実行列ヘッダーが環境マトリクスと不一致**: 観点シートで選定した環境組み合わせ(TH単体, TK+TI+TD等)をそのままTC列ヘッダーに反映すべき
  6. **リグレッション観点が画面パスレベルまで具体化されていない**: 「TH 請求書一覧 > 請求書詳細 添付ファイルタブ」のようにパンくず形式で記載すべき
- **ルール化**: 一部済 → 記述ルール.md セクション6-7, 観点チェックリスト セクション5
