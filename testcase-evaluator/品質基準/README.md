# テスト品質基準 (Single Source of Truth)

テストケース・テスト観点の品質に関する基準を一元管理するフォルダです。
評価ツール・生成ツール・Skillはすべてこのフォルダを参照します。

---

## ファイル構成

| ファイル | 内容 | 主な参照元 |
|---------|------|-----------|
| `記述ルール.md` | 書式・表現・フォーマットの統一基準 | generate系スクリプト, testcase-gen Skill |
| `観点チェックリスト.md` | 16機能+7非機能テスト観点の判定基準 | 観点シート作成時, testcase-gen Skill |
| `テストパターン.md` | 6つの具体的テスト設計パターン | テストケース設計時 |
| `評価基準.md` | 自動評価の6チェック項目+採点ロジック | evaluate_testcases.py, testcase-eval Skill |
| `改善ログ.md` | 評価・レビューで得た知見の蓄積 | 次回以降の設計・評価時 |
| `テスト設計ワークフロー.md` | 5フェーズ一気通貫のテスト設計手順 | testcase-gen Skill |
| `プロダクト情報/` | TOKIUM各プロダクトの画面構成・固有テスト観点 | テスト設計時, testcase-gen Skill |

---

## 更新ルール

1. **変更時は必ずこのフォルダのファイルを更新する**（元ソースは参照用として残す）
2. 評価ツールのチェックロジック変更時は `評価基準.md` も同時更新
3. レビュー指摘で新知見を得たら `改善ログ.md` に追記
4. ガードレール原本（`仕様・要件/ガードレール/`）との差分が生じた場合はこちらを優先
