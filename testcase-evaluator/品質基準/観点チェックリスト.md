# テスト観点チェックリスト

テスト設計時に各観点の対象/対象外を判定するためのチェックリストです。
各項目を順に確認し、対象外の場合は理由を明記してください。

---

## 1. 機能テスト観点（16項目）

### 1-1. 仕様のチェック [必須]
- [ ] 仕様に記載の挙動が仕様通りに実装されていること
- [ ] 要件に記載されていることが実現されていること
- [ ] 正常系: 仕様想定の入力→仕様通りの結果
- [ ] 異常系: 想定外の入力→エラー理由表示 or 実行不可

### 1-2. 仕様の妥当性チェック [必須]
- [ ] 仕様通りでもユーザー目線で不親切な点がないか

### 1-3. 入力値テスト
- [ ] 正常値（ひらがな・全角カタカナ・半角カタカナ・数値・マイナス・小数・漢字・全角英字・半角英字）
- [ ] 境界値（最小・最大・最小-1・最大+1）
- [ ] 異常値（型違い・空・null・特殊文字・SQLi・XSS）
- [ ] 桁数制限（上限ぴったり・上限+1）
- [ ] 禁則文字・フォーマット制約
- [ ] **金額入力欄は必ずP1でテストケース化**

### 1-4. 認証・認可
- [ ] ログイン正常系（メール認証・SSO・SAML・代理ログイン・テナント切替）
- [ ] ログアウト正常系
- [ ] セッションタイムアウト
- [ ] 権限のないページへの直接URLアクセス
- [ ] ロール別の表示・操作権限の違い
- [ ] 複数タブ・複数デバイスでの同時ログイン
- [ ] ログインセッション維持（既存SaaSセッション→AIエージェントログイン時のリダイレクト）
- [ ] 不正アクセスの検知（ログ記録）
- [ ] 権限定義通りの操作制限

### 1-5. データ整合性
- [ ] CRUD一連フロー（登録→参照→更新→削除）
- [ ] 関連データへの波及確認
- [ ] 同時更新（排他制御）
- [ ] トランザクション途中エラー・ロールバック

### 1-6. 画面遷移・UI
- [ ] 正常な画面遷移フロー
- [ ] ブラウザバック・リロードの挙動
- [ ] 二重送信防止
- [ ] バリデーションエラー時のメッセージ表示
- [ ] 必須項目未入力

### 1-7. 高頻度
- [ ] ボタン連続押下（連続実行不可 or 重複実行されないこと）
- [ ] 操作の競合（複数ユーザー同一リソース同時操作）

### 1-8. 英語化対応
- [ ] 新規作成画面の英語化
- [ ] 変更箇所の英語化
- [ ] ボタン・文言・エラーメッセージの英語化

### 1-9. 外部連携API
- [ ] API経由の情報取得
- [ ] API経由の情報登録
- [ ] 既存外部連携APIの機能維持
- [ ] API仕様変更後の機能維持

### 1-10. 契約変更
- [ ] オプション契約状況に応じた表示変化
- [ ] 操作不可能な機能のバリデーション
- [ ] 全契約形態への影響確認

### 1-11. 既存機能への影響確認
- [ ] 新機能追加・修正で既存機能の挙動が維持されていること
- [ ] 影響範囲の詳細は「3. 影響範囲チェックリスト」を参照

### 1-12. シナリオテスト
- [ ] ユースケースに沿ったシナリオで不便・エラーがないこと

### 1-13. 金額確認
- [ ] 手入力/自動入力(OCR等)/自動仕訳/定期支払の金額計算
- [ ] 整数・負の値・小数それぞれの正確性
- [ ] 丸め誤差（発生する/しないケース両方）
- [ ] 割合按分の正確性（合計一致確認）
- [ ] 消費税額・源泉徴収税額の自動計算
- [ ] 税抜/税込切替での金額変動
- [ ] **金額計算関連の修正は必ずP1で網羅確認**

### 1-14. 時間経過系
- [ ] 日跨ぎ・月跨ぎ・年跨ぎ・年度跨ぎ
- [ ] 指定時刻経過後のステータス変化
- [ ] 長期間経過後の機能維持

### 1-15. リリースによるデータ移行の影響確認
- [ ] 設定値の意図せぬ変更なし
- [ ] リリース前の設定・データが操作可能
- [ ] 会計フォーマットの継続利用
- [ ] ロールバック時の復帰
- [ ] リリース跨ぎの途中状態データ（申請中→承認前にリリース）

### 1-16. 互換性確認（ファイルフォーマット）
- [ ] 利用可能フォーマットの処理可否
- [ ] 対象外フォーマットのエラー表示
- [ ] 各拡張子（jpg, png, pdf, csv, xlsx等）での正常動作
- [ ] フォーマット変更時の既存ファイル互換

## 2. 非機能テスト観点（7項目）

### 2-1. パフォーマンス・負荷
- [ ] 大量データ時の応答速度
- [ ] 同時アクセス時の挙動
- [ ] タイムアウト設定値
- [ ] 大容量/大量ファイルのアップロード
- [ ] インポートの大量処理
- [ ] マスタ件数膨大時の挙動

### 2-2. 互換性
- [ ] 対応ブラウザ（Chrome/Edge/Safari/Firefox最新）
- [ ] 画面サイズ（PC/タブレット/スマートフォン）
- [ ] 対応OS（Windows/mac/Android/iOS）
- [ ] アプリ版での同等機能

### 2-3. セキュリティ
- [ ] 機密情報のURLパラメータ露出
- [ ] エラーメッセージへの内部情報露出
- [ ] ログへの個人情報出力

### 2-4. UX
- [ ] 画面崩れなし
- [ ] サイズ変更可能領域の調整後の崩れ
- [ ] ユースケースに沿った操作性（無駄な視線・マウス移動なし）
- [ ] 画面説明の十分さ（経験の少ないユーザーでも理解可能）
- [ ] 既存画面からの変化の大きさ

### 2-5. 統一性
- [ ] 複数プロダクト間のデザイン統一性
- [ ] 複数プロダクト間の操作感統一性
- [ ] Web-スマホアプリ間の操作感統一性
- [ ] 文言表記の統一

### 2-6. お客様提供ドキュメント
- [ ] ヘルプ文言のわかりやすさ
- [ ] マニュアルの正確性（画面キャプチャ・手順が最新仕様と一致）

### 2-7. 本番環境固有の確認
- [ ] 本番環境固有のオブジェクトや機能への影響
- [ ] サポート用チャットボタン・通知バナー等の正常動作

## 3. 影響範囲チェックリスト

1-11「既存機能への影響確認」実施時に、変更の種類に応じて確認する。

### 3-1. 画面内に項目追加がある場合
- [ ] 画面全体の挙動維持
- [ ] 表示崩れなし
- [ ] リリース前データの利用・更新可能
- [ ] 追加コンポーネントの他項目の挙動維持
- [ ] APIでの情報取得・設定

### 3-2. 計算ルール変更がある場合
- [ ] 演算パターン網羅確認
- [ ] 会計データ出力の数値正確性

### 3-3. 経費データ変更がある場合
- [ ] 全パターンの新規作成（手動/自動/カード連携）
- [ ] 経費統合の正常動作
- [ ] オプション設定（全ON/全OFF/抜粋ON）
- [ ] 添付画像（対応拡張子/表裏/削除/入替/タイムスタンプ）
- [ ] 変更履歴の全項目記録
- [ ] APIとアプリでの操作

### 3-4. 請求書データ変更がある場合
- [ ] 全パターンの新規作成（手動/一括/自動/移動）
- [ ] 関連書類設定
- [ ] オプション設定
- [ ] 添付画像（10枠目まで）
- [ ] 明細一覧操作（追加/削除/変更/移動/インポート/過去明細）
- [ ] 変更履歴・定期支払・APIとアプリ

### 3-5. 国税関係書類データ変更がある場合
- [ ] 全パターンの新規作成
- [ ] 関連書類設定・オプション設定
- [ ] 添付画像・変更履歴・API

### 3-6. 権限変更がある場合
- [ ] ユーザーの新規作成・変更（権限個別/全権限/抜粋）
- [ ] 既存ユーザーの権限維持・不正操作不可
- [ ] 設定変更後の挙動
- [ ] API

## 4. プロダクト固有の注意ポイント

### よく見落とすポイント
- アプリ版の影響確認（経費作成/申請/承認、支払依頼承認、カード連携）
- マスタ更新・削除時の過去データへの影響（例: プロジェクト削除→過去経費データの破損）
- 入力フィールド追加時の検索画面・集計画面・会計データ定期出力への波及

### 過去バグから得た観点
- 経理プロダクトのため金額計算は最重視
- 小数点切捨て/切上げ、割合按分はバグ発生頻度・緊急度ともに高い
- 日付入力は和暦・西暦混在に注意

### 外部連携
- 画面上の検索条件追加 → API側のパラメータ追加も確認
