# テストケース自動評価基準

`evaluate_testcases.py` による100点満点の自動評価チェック項目と採点ロジックです。

---

## 評価フロー

```
評価対象.xlsx ──→ evaluate_testcases.py ──→ 100点満点レポート
                       │
                  benchmark.json（学習済み基準値、チェック5で使用）
```

---

## チェック項目一覧

| # | チェック項目 | 上限減点 | 1件あたり | 根拠ルール |
|---|-------------|---------|----------|-----------|
| 1 | 期待値の曖昧表現 | -20点 | -3〜-5点 | 曖昧に書かない |
| 2 | 期待値の複数混在 | -15点 | -2点 | 1ケース1期待値 |
| 3 | 我流の単語 | -15点 | -1点 | 仕様書の用語に統一 |
| 4 | 手順の省略 | -10点 | -2点 | 手順は極力省かない |
| 5 | 粒度・品質 | -15点 | (複合) | 参考例との比較 |
| 6 | ガードレール準拠 | -10点 | (複合) | 金額P1+観点TC整合 |

---

## チェック1: 期待値の曖昧表現（上限 -20点）

期待値セルに含まれるNGパターンを検出する。

| パターン | 減点 |
|---------|------|
| `正常であること` / `正常に動作すること` | -5点 |
| `問題ないこと` / `問題なく〜こと` | -5点 |
| `仕様通りであること` / `仕様通りに動作すること` | -5点 |
| `正しく動作すること` | -3点 |
| `適切に表示されること` / `適切であること` | -3点 |
| `正しく表示されること` | -3点 |
| `期待通り` / `想定通り` | -3点 |

### 偽陽性除外パターン
- 「スペルと英語が適切であること」→ 英語化チェック文脈では具体的
- 「正しくありません」→ エラーメッセージの引用
- 「正しく入力」→ バリデーション文脈

---

## チェック2: 期待値の複数混在（上限 -15点 / 1件 -2点）

期待値セル内に「こと」が2回以上含まれる場合に検出。

**括弧内の「こと」は除外**: `strip_parens()` で括弧内の補足コメントを除外してからカウント。

---

## チェック3: 我流の単語（上限 -15点 / 1件 -1点）

NGワードファイル（TSV: `NGワード<TAB>正式名称`）で定義した略語・造語を検出。

**チェック対象**: 確認すること、画面、確認対象、詳細、手順、期待値、備考 + 観点シート全セル

---

## チェック4: 手順の省略（上限 -10点 / 1件 -2点）

同じ「確認すること」カテゴリ内で**全ケースが手順空欄**の場合のみ検出。
（グループ内の先頭に手順があり後続が空欄の「グループ共有手順」パターンは許容）

---

## チェック5: 粒度・品質（上限 -15点）

`benchmark.json`（参考例から学習した基準値）と比較して評価。

| 観点 | 配点 | 判定基準 |
|------|------|----------|
| 期待値の長さ | 5点 | ideal_min未満/ideal_max超が20%超→-5, 10%超→-3, 5%超→-1 |
| 手順の詳細度 | 5点 | 記載率60%未満→-5, 80%未満→-3。番号付きステップ率20%未満→-2 |
| 末尾表現 | 5点 | 良好末尾率が基準-10%未満→-5, 基準未満→-3, 基準+10%未満→-1 |

benchmark.json が無い場合はデフォルト値で動作。

---

## チェック6: ガードレール準拠（上限 -10点）

| 観点 | 減点 | 判定基準 |
|------|------|----------|
| 金額P1設定 | -5点 | 金額関連テストケースが存在するのにP1が設定されていない |
| 観点TC整合 | -5点 | 観点シートの対象項目にテストケースが紐づいていない |

---

## スコアの目安

| スコア | 評価 |
|--------|------|
| 100点 | 運用ルール完全準拠 |
| 90-99点 | 軽微な修正で対応可能 |
| 80-89点 | 複数の修正が必要 |
| 70-79点 | 用語統一や期待値の書き直しが必要 |
| 70点未満 | 全体的な見直しが必要 |

---

## benchmark.json の主要構造

```json
{
  "_meta": { "source_files": [...], "total_cases": 6126 },
  "expected_length": { "avg": 21.1, "p5": 9, "p95": 39, "ideal_min": 9, "ideal_max": 80 },
  "steps_coverage": { "rate": 0.770, "good": 0.80, "warning": 0.60 },
  "single_koto_rate": { "rate": 0.989, "good": 0.98 },
  "good_endings": { "rate": 0.910, "good": 0.80, "patterns": ["されていること", ...] }
}
```

学習データの追加: `参考例/` にExcelを追加 → `learn_benchmark.py` を再実行 → benchmark.json 更新
